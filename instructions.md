# Instructions

## Formaal

Byg en lille, statisk hjemmeside med spillet kryds og bolle, hvor brugeren spiller mod computeren.

## Krav

- Brug ren HTML, CSS og JavaScript.
- Spillet skal koere direkte i browseren uden backend.
- Designet skal have et gammeldags look.
- Brugeren skal kunne starte et nyt spil uden at reloade siden.
- Computeren skal automatisk foretage sit traek efter spilleren.

## Designretning

- Vaelg en tydelig retro stil frem for et moderne standardlook.
- Brug varme eller afdampede farver frem for generiske standardfarver.
- Brug kraftige borders, paneler og enkel visuel dybde.
- Typografi maa gerne foeles klassisk eller arkade-inspireret.
- UI skal stadig vaere laeseligt og enkelt paa mobil.

## Teknisk retning

- Hold koden enkel og opdelt efter ansvar.
- HTML skal kun indeholde struktur og indhold.
- CSS skal styre udseende, spacing og responsive regler.
- JavaScript skal styre state, rendering og computerens beslutninger.
- AI maa gerne implementeres med minimax, da spillet er lille nok til det.

## Forventet filansvar

- `index.html`: titel, intro, statusfelt, spillebraet og kontrolknapper
- `styles.css`: farver, layout, typografi, animationer og responsive regler
- `game.js`: board-state, spillertraek, AI-traek, vinderlogik og reset

## Acceptkriterier

- Man kan spille en hel omgang mod computeren.
- Computeren laver kun lovlige traek.
- Spillet stopper korrekt ved sejr eller uafgjort.
- Nyt spil nulstiller braettet og statusfeltet.
- Siden har et synligt gammeldags visuelt udtryk.

## GitHub

Projektet boer versionsstyres med git og pushes til GitHub, saa snart basisfilerne er paa plads.
